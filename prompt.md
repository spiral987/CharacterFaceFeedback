# Project: Character Illustration Diagnostic & Reflection System

## 1. プロジェクトの目的と概要 (Project Overview)
本プロジェクトは、二次創作やキャラクターイラストの制作途中（WIP）において、描き手が自身の「絵柄（エージェンシー）」を保ちつつ、公式資料画像との「似せ方」や「雰囲気」のズレを客観的に把握するためのCreativity Support Tool (CST) である。

システムによる「自動修正」は行わず、Human-in-the-Loopのベイズ最適化（ギャラリーベースの一対比較）を用いて、「なぜ似ていないのか」というパラメータのズレ（Delta）を特定し、描き手に「診断書」として提示することで、キャンバス上での深いリフレクション（内省）と学習を支援する。

## 2. 技術スタック (Tech Stack)
* **Frontend Framework:** Next.js 16 (App Router), React 19
* **Styling:** Tailwind CSS
* **Language:** TypeScript
* **Image Processing (Browser):**
    * `ag-psd` (ブラウザ上でのPSDファイルのパース、レイヤー抽出)
    * HTML5 Canvas API / WebGL (抽出したレイヤーの独立した描画とアフィン変換)
* **Backend/Optimization (将来構想):** Python (FastAPI), BoTorch/GPyOpt等のベイズ最適化ライブラリ
* **Database/Storage:** Supabase or Firebase (セッションデータ、抽出レイヤー画像の保存)

## 3. コアワークフロー (Core Workflows)

### Step 1: 描き手によるアップロードと意図の定義 (Creator Setup)
* ユーザーは制作途中のイラスト（PSD形式）と、目標とする公式の資料画像（JPG/PNG）をアップロードする。
* **制約:** PSDファイルは特定の命名規則（例: `Eye_L`, `Face`）に従ってラスタライズされたレイヤー構造を持つ必要がある。
* フロントエンド（`ag-psd`）でPSDを解析し、対象レイヤーの画像データを抽出・検証する。
* ユーザーは「少し大人っぽくしたい」などの意図（Goal）をテキストで入力し、評価用セッションリンク（共有URL）を生成する。

### Step 2: 評価者による一対比較 (Interactive Evaluation / HITL)
* 評価者が共有URL（`/evaluate?session=...`）にアクセスする。
* システムは、顔パーツ（目、輪郭など）のアフィン変換パラメータ（移動、回転、拡大縮小）から意味のある2次元探索平面を切り出し、複数のバリエーション画像をグリッド状にCanvasレンダリングして提示する。
* 評価者は資料画像と「意図」を見比べ、「最も資料/意図に近いバリエーション」を選択（クリック）する。
* この選択を観測値としてベイズ最適化ループを回し（10〜20回程度）、最適解へ収束させる。

### Step 3: 診断結果とリフレクション (Diagnosis & Reflection)
* 最適化が完了すると、描き手側に「診断書」が提示される。
* 最適化された画像をそのまま見せるのではなく、「左目をX軸に+5%移動、輪郭を-2%縮小」といった具体的なパラメータ差分（Delta）と、視覚的なヒートマップを提示する。
* 描き手はこの客観的データをもとに、自身のペイントツール上で修正を行う。

## 4. 現在の開発フェーズとAIへの指示 (Current Phase & Instructions for AI)
**現在のフェーズ: フェーズ1（PoC）- PSD解析とブラウザ上での独立アフィン変換基盤の構築**

AIアシスタントは、以下のルールに従ってコードを生成・修正すること：
1.  **ブラウザ完結の画像処理:** サーバーコストと遅延を防ぐため、PSDのパースと画像の変形処理は可能な限りフロントエンド（クライアントサイド）のJavaScript/Canvasで行うアーキテクチャを優先すること。
2.  **モダンなReact実装:** 既存のNext.js App Routerの規約に従い、クライアントコンポーネント（`"use client"`）を適切に分離してCanvas操作を実装すること。
3.  **UI/UXのトーン:** クリエイター向けのツールであるため、UIはノイズを減らし、アップロードされた画像やキャンバスが主役になるようなミニマルで洗練されたTailwind CSSのスタイリングを心がけること。
